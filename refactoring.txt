
это план рефакторинга:
Цель - упрощение, оптимизация и UX
1. существующие проекты в solution остаются как есть
2. добовляются новый wpf проект и новый резолвер. В номых проектах все классы находятся в отдельных файлах. Бизнес-логика в .xsml.cs только для взаимодействия с UI.
2.1 название Yara (yet another russian accenter) (Яра)
2.2 namespace WhiteBehemoth.Yara и WhiteBehemoth.Resolver
Приложение всё также имеет одну главную форму. В ней есть табы "Текст" и "Настройки". "Текст" - бывший "Результат". "Оригинал" убирается. Главное меню - сохраняется. 
Когда программы открывается, на месте текста на ходится плейхоледер текст - краткая инструкция, которая исчезает в открытия файла или вставки/ввода текста
Когда пользователь открывает файл, он отображается на закладке "Текст". Статистика считается как есть сейчас.
Когда начинается разрешение омографовв, текст переходит в read-only (а не parent-tab disabled как сейчас).
Меняется логика разрешения омографов. В резолвер передаётся не текст целиком, а предложение с омографом.
Текст на форме обновляется по ходу разрешения омографом, скролируясь на текущее предложение. в статус баре отображается прогресс разрешения омографов, включая returned reasoning.
Если пользователь прервал процесс, readonly снимается, текст остаётся с теми омографами, которые были разрешены на момент остановки. 
Меняется принцип навигации по "неуверенным омографам". Пользрватель может менять порог уже после разрешения омографов, и навигация будет работать по этому порогу. 
Когда пользователь делает выбор по Alt+цифра, вероятность повышается до 1 (как сейчас). Если пользователь нажал Alt+`, вероятность повышается до 1 без изменения омографа.
Items в комбобокс "очистки текста" по умочанию checked.
Поиска и замена переносятся на отдельный toolbar, скрытый по умолчанию, который появляется при нажатии Ctrl+F (новое меню добавить).

Настройка меняется следующим образом:
1. Убрать всё по TTS (команду меню на озвучку - тоже)
2. в LLM секции вместо двух настроек OpanAI и FoundryLocal, сделать возможность добавлять несколько OpenAI configs (разные URL/model names/keys).
для foundrylocal - видель какие мождели скачены, какие нет.
Пользователь может выбрать один из OpenAI конфигов или foundrylocal с конкретной моделью.
Сделанный выбор (а так же текущий выбор из настроек при открытии программы) отображается в статус-баре. "LLM: BaseURL - model name" (на месте где потом будет reasoning по омографиам)
